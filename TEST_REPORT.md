# KOH_APP テスト報告書（提出用）

## 1. 報告書の目的
本報告書は、KOH_APP の現行実装に対する確認内容を整理し、実装済み機能と未対応機能を明確化することを目的とする。

## 2. 対象バージョン / 前提
- 対象: 現在の `main` 作業ツリー
- アプリ種別: Spring Boot 学習用予約アプリ
- 予約上限: 同一日付・同一時間枠あたり 10 名
- `/reservations`: 一覧表示は対象外（`/thanks` へリダイレクト）

## 3. 実施観点と結果

### 3.1 正常登録
- 観点: 有効な入力で予約が保存され、`/thanks` に遷移する
- 結果: OK

### 3.2 バリデーション
- 観点: 必須、形式、文字数の各チェックが機能する
- 主な確認対象:
  - 必須入力（予約日、時間枠、人数、名前、メール、電話）
  - メール形式
  - 電話番号形式
  - 文字数（名前50、メール200、備考300）
- 結果: OK

### 3.3 定員超過
- 観点: 同一枠の合計人数が 10 を超える場合に予約を拒否し、入力画面へ戻る
- 結果: OK

### 3.4 例外ハンドリング
- 観点: 想定外例外発生時にグローバル例外ハンドラでエラーページを返し、詳細を利用者へ露出しない
- 結果: OK

### 3.5 同時送信対策
- 観点: 同一枠への並列リクエストで、定員超過が成立しない
- 主な確認ケース:
  - 競合時に成功/失敗が分岐すること
  - 最終合計人数が 10 以下であること
  - 別時間枠は干渉しないこと
- 結果: OK

## 4. 対象外 / 未対応
- 予約一覧表示機能（`/reservations` での一覧画面表示）
- 認証 / 認可（ログイン、権限制御）
- 管理者向け運用機能

提出時点の方針として、本アプリは公開画面のみを対象としており、管理機能を含まないため認証 / 認可は未実装とする。
将来、管理画面や管理APIを追加する段階で導入する。

## 5. 実行ベース
- 自動テスト: `mvn test`
- 補足: 現行コードでは予約フロー確認テストおよび同時送信確認テストを実装済み

## 6. 結論
提出物として、現行実装の対象範囲（予約入力、バリデーション、定員制御、例外時表示、同時送信対策）に対する確認内容は整合している。
未実装機能は「対象外 / 未対応」として明示した。
