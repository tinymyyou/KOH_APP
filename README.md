# KOH_APP

KOH_APP の予約機能を実装した Spring Boot アプリです。  
ワークショップ予約を題材にした学習用・提出用の最小実装です。

## アプリ概要
- 予約入力フォームから予約を受け付ける Web アプリ
- 入力値のバリデーションと定員チェックを実施
- 正常時は完了画面へ遷移

## 使用技術
- Java 17
- Spring Boot 3.3.8
- Thymeleaf
- Spring Data JPA
- Bean Validation
- H2 Database
- Maven

## 主な機能
- 予約入力（`/reserve`）
- 必須チェック
- メール形式チェック
- 電話番号形式チェック
- 文字数制約
  - 名前: 50文字以内
  - メールアドレス: 200文字以内
  - 備考: 300文字以内
- 定員チェック（同一日付・同一時間枠で上限10名）
- 予約成功時は `redirect:/thanks`

同時送信対策として、同一の予約日・時間枠に対する予約処理は予約枠単位で排他制御し、定員判定と保存を直列化しています。これにより同時リクエスト時でも上限10名を超える予約成立を防ぐ構成としており、並列実行テストで確認済みです。

## 画面 / ルーティング
- `GET /` : トップページ
- `GET /faq` : FAQ ページ
- `GET /reserve` : 予約入力ページ
- `POST /reserve` : 予約送信
- `GET /thanks` : 予約完了ページ
- `GET /reservations` : `redirect:/thanks`（一覧表示は対象外）

## 起動方法
### 必要環境
- JDK 17
- Maven 3.9 以降

### 起動コマンド
```bash
mvn spring-boot:run
```

### アクセス URL
- `http://localhost:8080/`

通常起動時は `src/main/resources/application.properties` を使用し、提出物としての通常動作確認を想定した設定（ファイルベース H2、`ddl-auto=update`、SQLログ抑制、H2 Console 無効）で実行します。これは日常の起動・確認用の設定です。

テスト実行時は `src/test/resources/application.properties` が適用され、`jdbc:h2:mem:` のメモリDBと `ddl-auto=create-drop` を使ってテストごとに独立したDB状態で検証します。テストデータを実運用用データへ混在させず、毎回同じ前提で再現性を保つために設定を分けています。

## テスト・確認観点
- 正常系: 有効な入力で `/thanks` に遷移
- 必須チェック: 未入力時に入力画面へ戻る
- 形式・文字数チェック: 不正値入力時にエラー表示
- 定員チェック: 同一枠の合計人数が 10 名を超える予約を拒否
- 同時送信対策: 同一枠の並列予約でも上限超過が成立しないこと

## 補足
- 一覧表示機能は現在の対象外（`/reservations` は完了画面へリダイレクト）
- 学習用実装のため、本番運用向けの認証・権限制御などは未対応

## 認証 / 認可方針（提出時点）
- 本提出物は公開画面（`/`, `/faq`, `/reserve`, `/thanks`）のみを対象とする
- 管理画面・管理APIは対象外のため、認証 / 認可は実装しない
- 将来、管理機能を追加する場合に Spring Security 等で認証 / 認可を導入する
